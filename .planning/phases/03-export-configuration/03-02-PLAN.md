The button calls a function `exportCSV()` that:
- Takes the current `orders` array from state (the currently displayed/filtered page data). Per D-03, this is client-side generation from currently loaded data.
- If the user wants ALL delayed orders (not just current page), first fetch all with a large per_page: `fetch('/api/orders?filter=delayed&per_page=9999')`. Use this approach: if `filter === 'delayed'`, fetch all delayed orders for export; otherwise export current page data.
- Builds CSV string with pt-BR headers per D-02: `ID Pedido,Nome Cliente,Codigo Rastreio,Pais,Dias em Transito,Threshold,Status,Data Envio`
- Maps each order to a CSV row. For dates, format as DD/MM/YYYY. Escape commas in values by wrapping in double quotes. Use comma as delimiter.
- Creates Blob with BOM prefix (`\uFEFF`) for Excel pt-BR compatibility, generates download link via `URL.createObjectURL`, triggers click, revokes URL.
- Filename per D-04: `pedidos-atrasados-YYYY-MM-DD.csv` using current date.

Style the button to match existing filter bar aesthetic (same padding, border-radius, font-size as filter buttons). Use a subtle green/accent color to distinguish it as an action button.
