5. Keep the CRON_SECRET auth check on the POST handler (existing pattern: `req.headers.get('x-cron-secret') !== process.env.CRON_SECRET`)

The sync pipeline has 3 stages:

**Stage 1: Fetch orders and shipments in parallel**
Call `fetchAllOrders()` and `fetchAllShipments()` via `Promise.all`. Both functions already handle mock fallback internally (per D-10), so the sync route does not need mock logic.

**Stage 2: Build shipment lookup and upsert orders**
- Create a Map from shipments: key = `order_id` (number), value = array of ShipoffersShipment
- For each ShipoffersOrder:
  - Look up shipment(s) by `order.id` in the map
  - Get `tracking_number` from the FIRST matched shipment, or null if no shipments
  - Get `shipped_at` from the first matched shipment, parse to Date, or null
  - Calculate `daysInTransit`: if shipped_at exists, `Math.floor((Date.now() - shippedAt.getTime()) / 86400000)`, else null
  - Get `delayThreshold` via `getDelayThreshold(order.shipping_address?.country)`
  - Upsert into prisma.order with these EXACT field mappings (per API-04):
    - `shipofffersId` = `String(order.id)` (note triple-f typo in schema -- KEEP IT as-is)
    - `trackingCode` = first shipment's `tracking_number` or null
    - `customerName` = `order.shipping_address?.name` or null
    - `customerEmail` = `order.email` or null
    - `destinationCountry` = `order.shipping_address?.country` or null
    - `shippedAt` = parsed Date from shipment's `shipped_at`, or null
    - `daysInTransit` = calculated int or null
    - `delayThreshold` = from delay rules

**Stage 3: Track packages via 17track (per API-05, D-04, D-11)**
- Query all orders that have a `trackingCode` (not null) and status NOT 'DELIVERED':
  `prisma.order.findMany({ where: { trackingCode: { not: null }, status: { not: 'DELIVERED' } }, select: { id: true, trackingCode: true, daysInTransit: true, delayThreshold: true } })`
- Extract tracking codes (filter out nulls), call `trackPackages(codes)`
- Create a Map from TrackingResult[] keyed by `trackingCode`
- For each trackable order, find its result and update:
  - `status` = result.status
  - `deliveredAt` = result.deliveredAt
  - `lastTrackingSync` = `new Date()`
  - `rawTrackingData` = result.rawData
  - `isDelayed` = status is NOT 'DELIVERED' AND daysInTransit is not null AND daysInTransit > delayThreshold

**Response format:** Return `Response.json({ success: true, orders: ordersCount, shipments: shipmentsCount, tracked: trackedCount })` using the native Response (not NextResponse, per Next.js 16 docs).

**Error handling:** Wrap entire pipeline in try/catch. Log with `console.error('[SYNC ERROR]', error)`. Return `Response.json({ error: 'Sync failed' }, { status: 500 })`.

| Cron/User -> Sync API | Protected by CRON_SECRET header check |

| Sync API -> 17track | 17token header over HTTPS |
| Sync API -> SQLite DB | Local file, no network boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Spoofing | POST /api/sync | mitigate | CRON_SECRET header check rejects unauthorized sync triggers |
| T-01-05 | Denial of Service | Pagination loop | mitigate | per_page capped at 250; loop breaks on empty/short response; 17track chunked to 40 per request |
| T-01-06 | Information Disclosure | Error responses | mitigate | Generic "Sync failed" to client; detailed errors only in server console.error logs |
