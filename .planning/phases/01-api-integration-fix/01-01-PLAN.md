  <action>
Completely rewrite lib/shipoffers.ts. The new implementation must follow these decisions:

**Authentication (per D-01):** HTTP Basic Auth via Authorization header. Construct the header as `Basic ${Buffer.from(user + ':' + pass).toString('base64')}` where `user` = `process.env.SHIPOFFERS_API_USER` and `pass` = `process.env.SHIPOFFERS_API_PASS`. Do NOT use api_key as query param -- Swagger UI is misleading; the real API returns `www-authenticate: Basic realm="API Authorization"`.

**Mock mode detection (per D-10):** The client falls back to mock data in TWO cases:
- `process.env.USE_MOCK === 'true'` (explicit toggle)
- `SHIPOFFERS_API_USER` or `SHIPOFFERS_API_PASS` are empty/undefined (credentials not yet available)
Log which fallback reason triggered: `[MOCK] USE_MOCK=true` or `[MOCK] Shipoffers credentials not configured`.

**Base URL:** `process.env.SHIPOFFERS_API_URL` which is `https://api.shipoffers.com`.

**Endpoints (per D-05, D-06, D-09):**
- Orders: `GET /api/stores/${storeId}/orders.json`
- All shipments: `GET /api/stores/${storeId}/shipments.json`
- Per-order shipments: `GET /api/stores/${storeId}/orders/${orderId}/shipments.json` (per D-07)
- Single order: `GET /api/stores/${storeId}/orders/${orderId}.json` (per D-08)
Note the `.json` suffix on ALL endpoints per D-09. The `/api/` prefix is part of the path, not the base URL.

**Store ID:** Read from `process.env.SHIPOFFERS_STORE_ID`.

**Pagination:** Use `page` and `per_page` query params. Default `per_page` to 250. Loop: fetch page 1, if response array length equals per_page, fetch next page. Stop when shorter or empty.

**Exported functions:**
- `fetchAllOrders(params?)` -- paginates through /orders.json, returns ShipoffersOrder[]
- `fetchAllShipments(params?)` -- paginates through /shipments.json, returns ShipoffersShipment[]
- `fetchOrderShipments(orderId)` -- single order's shipments (per D-07)

**Exported interfaces** (field names based on Swagger -- exact fields will be confirmed when API responds, but use reasonable defaults):
- `ShipoffersOrder` with fields: id (number), order_number (string), email (string), shipping_address (object with name, country, city, etc.), created_at (string), updated_at (string), status (string)
- `ShipoffersShipment` with fields: id (number), order_id (number), tracking_number (string), carrier (string), shipped_at (string), created_at (string), updated_at (string)

**Axios client setup:** Create with baseURL from env, set Authorization header with Basic Auth. Use Content-Type application/json.

**Keep importing from mock-data.ts** for the mock fallback path, same as current code does.
